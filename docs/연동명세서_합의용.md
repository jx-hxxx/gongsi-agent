# AI ↔ 백엔드 연동 명세서 (합의용)

> 두 팀이 **맞춰야 하는 인터페이스 계약**만 담음. (내부 실행·DB·파이프라인은 제외)
> Spring Boot → FastAPI, **동기 JSON**, 단일 엔드포인트.

| 항목 | 값 |
|---|---|
| Method / Path | `POST /api/v1/chat` |
| 포트 | 8001 (사설망, 앱 인증 없음) |
| Content-Type | `application/json; charset=utf-8` (요청·응답) |
| 헤더 | `X-Trace-Id`(UUID, 권장) → 응답에 echo |

---

## 1. 요청 (Spring → FastAPI)

| 필드 | 타입 | 필수 | 설명 |
|---|---|---|---|
| `roomId` | integer | ✅ | 대화방 ID (응답에 echo) |
| `userSeq` | integer | ✅ | 사용자 시퀀스 (로그용) |
| `companyContext.corpCode` | string | ✅ | DART 고유번호 8자리 (검색 범위 키) |
| `companyContext.corpName` | string | ✅ | 기업명 |
| `messages` | array | ✅ | 멀티턴. 시간순, **마지막=`role:user`(현재 질문)** |
| `messages[].role` | enum | ✅ | `user` \| `assistant` |
| `messages[].content` | string | ✅ | 본문 |

```json
{
  "roomId": 142,
  "userSeq": 12345,
  "companyContext": {"corpCode": "00126380", "corpName": "삼성전자"},
  "messages": [
    {"role": "user", "content": "영업이익 알려줘"},
    {"role": "assistant", "content": "..."},
    {"role": "user", "content": "작년보다 얼마나 늘었어?"}
  ]
}
```

---

## 2. 응답 (FastAPI → Spring)

모든 키 항상 존재(값은 채움/`null`). **추론 실패도 HTTP 200 + `error`**, 시스템 실패만 4xx/5xx.

| 필드 | 타입 | 설명 |
|---|---|---|
| `roomId` | integer | 요청 echo |
| `intent` | string\|null | `qa`/`summary`/`macro`/`smalltalk`/`out_of_scope` |
| `answerText` | string\|null | 답변 본문 (완전 실패 시 `null`) |
| `sourceContent` | string\|null | 인용 근거 텍스트(§4 포맷). 없으면 `null` |
| `macroSnapshot` | string\|null | 거시지표 텍스트. 미사용 `null` |
| `sources` | array\|null | 구조화 출처 (`rceptNo,reportNm,rceptDt,sectionTitle,quote,score`) |
| `outOfScope` | boolean | true → 다른 회사 질문 → **프론트 팝업** |
| `detectedCompany` | string\|null | outOfScope 시 감지 회사 |
| `needsClarification` | boolean | true → `answerText`가 **되묻는 질문** |
| `verification` | object\|null | `{verdict: pass/partial/fail, groundedScore: 0~1}` |
| `error` | object\|null | `{code, message, retriable}` (완전 실패 시) |

### 정상 예시 (qa)
```json
{
  "roomId": 142, "intent": "qa",
  "answerText": "연결 영업이익은 전기 32.7조 → 당기 43.6조, 10.875조 증가...",
  "sourceContent": "[20260310002820 / 삼성전자 2025 사업보고서 재무제표]\n영업이익 — 당기 43,601,051,000,000원...",
  "macroSnapshot": null,
  "sources": [{"rceptNo":"20260310002820","reportNm":"2025 사업보고서 재무제표","rceptDt":"20260310","sectionTitle":"재무제표 주요계정","quote":"영업이익 — 당기 43,601,051,000,000원...","score":1.0}],
  "outOfScope": false, "detectedCompany": null, "needsClarification": false,
  "verification": {"verdict":"pass","groundedScore":1.0},
  "error": null
}
```

### 다른 회사 예시 (out_of_scope)
```json
{
  "roomId": 142, "intent": "out_of_scope",
  "answerText": "이 방은 '삼성전자' 전용입니다. '현대차'는 해당 상세페이지에서 질문해 주세요.",
  "sourceContent": null, "macroSnapshot": null, "sources": null,
  "outOfScope": true, "detectedCompany": "현대차",
  "needsClarification": false, "verification": null, "error": null
}
```

### 실패 예시 (error)
```json
{
  "roomId": 142, "intent": "qa", "answerText": null,
  "sourceContent": null, "macroSnapshot": null, "sources": null,
  "outOfScope": false, "detectedCompany": null, "needsClarification": false,
  "verification": null,
  "error": {"code":"LLM_TIMEOUT","message":"...","retriable":true}
}
```

## 3. error 코드 / HTTP status

| error.code | 상황 | retriable |
|---|---|---|
| `EMBEDDING_FAILED` / `VECTOR_SEARCH_FAILED` | 임베딩·검색 실패 | true |
| `LLM_TIMEOUT` / `LLM_API_ERROR` / `RATE_LIMITED` | LLM 호출 문제 | true |
| `CONTEXT_OVERFLOW` / `CONTENT_FILTERED` | 컨텍스트 초과·필터 거부 | false |
| `INVALID_REQUEST` | 요청 스키마 오류 (HTTP 400) | false |
| `INTERNAL_ERROR` | 미분류 | true |

| HTTP | 상황 |
|---|---|
| 200 | 정상 + 추론실패(body.error) |
| 400 | 요청 스키마 오류(messages 마지막이 user 아님 등) |
| 503 / 504 | 서비스 다운 / 게이트웨이 타임아웃 |

## 4. `sourceContent` 포맷
```
[<rceptNo> / <corpName> <reportNm>]
<인용 본문>

[<rceptNo> / <corpName> <reportNm>]
<인용 본문>
```
청크 간 `\n\n`. (DB 저장은 이 문자열, 프론트 카드는 `sources[]`)

## 5. DB 연동 (맞춰야 할 부분)

**전제**: AI는 stateless → **대화 이력 DB는 백엔드 소유**. AI는 공시 코퍼스(벡터DB)만 보유하며 백엔드는 직접 접근하지 않음. 둘이 맞출 건 아래 2가지.

### 5.1 대화 이력 저장 — 우리 응답 → 백엔드 DB 컬럼 (제안)
백엔드 테이블(예 `tb_qa_history`)에 **AI 응답에서 저장할 필드**:

| 저장 컬럼(예) | ← 응답 필드 | 비고 |
|---|---|---|
| `room_id` | `roomId` | |
| `role` / `content` | 사용자질문 / `answerText` | 턴 본문 |
| `source_content` | `sourceContent` | 인용 근거 텍스트 그대로 |
| `macro_snapshot` | `macroSnapshot` | 거시 텍스트 (있을 때) |
| `intent` | `intent` | 분석용(선택) |
| `verdict` / `grounded_score` | `verification.verdict` / `.groundedScore` | 신뢰도(선택) |
| `created_at` | (백엔드 시각) | |

- 구조화 `sources[]`는 보통 저장 불필요(표시용). 필요 시 JSON 컬럼.
- **`error != null`(추론 실패) 응답은 이력에 저장하지 않음.**
- 테이블 DDL 자체는 백엔드가 자유롭게 설계 (위는 "이 데이터는 담아두면 좋다" 가이드).

### 5.2 검색 가능 기업(corpCode) 동기화
- **Spring의 선택 가능 회사 목록 = AI 코퍼스에 적재된 기업**과 일치해야 함.
- 현재 적재: **삼성전자 `00126380`, 현대자동차 `00164742`** (3개년 공시).
- 새 기업 추가 절차: **AI가 먼저 코퍼스 적재** → 그 다음 백엔드 목록에 추가. (적재 안 된 기업은 검색 비어 "확인 불가"만 나옴)
- 식별자는 **DART 8자리 `corpCode`로 통일** (백엔드가 회사명↔corpCode 매핑 보유).

## 6. 🤝 합의 확인 (백엔드 OK만 주면 확정)

AI측 제안 = 아래대로 구현됨. **다르면 의견 주세요.**

- [ ] **error 의미**: `error`는 *답을 못 만든 완전 실패*에만 채움. 부분 실패(검증/거시/재무 sub-step 실패하지만 답은 정상)는 `error` 안 쓰고 로그만.
- [ ] **검증 fail 시**: 답을 **반환**하고 `verdict:"fail"` + 낮은 score 표기 (거절 안 함). → 프론트가 "신뢰도 낮음" 표시.
- [ ] **messages 길이**: AI는 받은 것 중 **최근 10개 메시지만** 사용 (토큰 안전장치). 마지막 user는 항상 포함.
- [ ] **Response Timeout 90초**: CRAG 재검색·거시결합 겹치면 길어질 수 있어 90초 권장. (충분한지 확인)
- [ ] **`sources[]`**: 거시 합성 출처는 제외, 재무·공시 출처만 포함. `sourceContent`와 동일 내용의 구조화 버전.
- [ ] **DB 저장 컬럼**(§5.1): 위 매핑대로 백엔드가 `tb_qa_history` 등에 저장 — OK?
- [ ] **기업 목록 동기화**(§5.2): 선택 가능 회사 = 적재 기업(삼성·현대). 추가 시 AI 적재 선행 — OK?

> 기계판독 스키마: `chat_v2_schema.json` (요청·응답 JSON Schema, 코드 자동 생성)
