# 공시분석 AI (FastAPI) 실행·연동 가이드

> 백엔드가 FastAPI 서비스를 **띄우고 `POST /api/v1/chat`로 연결 확인**하기 위한 가이드.

---

## 1. 사전 준비

| 항목 | 값 |
|---|---|
| Python | 3.11 |
| 의존성 | `pip install -r requirements.txt` |
| 환경변수 | `.env` (아래) |
| 데이터 | `data/chroma/` (코퍼스 벡터DB) **필수** — §3 |

### 1.1 설치
```bash
python3.11 -m venv .venv
source .venv/bin/activate          # (Windows: .venv\Scripts\activate)
pip install -r requirements.txt
```

### 1.2 `.env` 작성 (`.env.example` 복사 후 키 채우기)
```dotenv
OPENAI_API_KEY=sk-proj-...      # ← 필수 (지현에게 받기)
DART_API_KEY=...                # ← 필수 (재무결합·코퍼스 적재용)
ECOS_API_KEY=sample             # 거시지표 (무인증 sample 가능, 실키 권장)
EMBEDDING_PROVIDER=openai
EMBEDDING_MODEL=text-embedding-3-small
CHROMA_DIR=./data/chroma
SQLITE_PATH=./data/app.db
```

---

## 2. 실행

```bash
uvicorn app.main:app --host 0.0.0.0 --port 8001
```

| 엔드포인트 | 용도 |
|---|---|
| `POST /api/v1/chat` | 챗 추론 (백엔드 연동 대상) |
| `GET  /health` | 헬스체크 (`{"status":"ok"}`) — 503 판정용 |
| `GET  /docs` | Swagger UI (스키마 확인) |
| `GET  /openapi.json` | OpenAPI 스펙 |

> 명세상 **포트 8001**, 사설망 내부 노출. 앱 인증 헤더 없음.

---

## 3. ⚠️ 코퍼스 데이터 (꼭 확인)

서비스가 답을 하려면 **Chroma 코퍼스(`data/chroma/`)가 있어야** 함. 없으면 검색이 비어 "확인 불가"만 나옴.

**두 가지 방법 중 택1:**
1. **데이터 전달받기** (권장·빠름) — 지현이 적재해둔 `data/chroma/`(약 1.2GB)를 받아 그 경로에 둠. 삼성전자·현대자동차 3개년 적재 완료 상태.
2. **직접 적재** — `DART_API_KEY` 세팅 후:
   ```bash
   python ingest_corpus.py          # 삼성전자·현대자동차 3개년 (수십 분, OpenAI 임베딩 비용 ~수백원)
   ```

> 적재·검색은 **반드시 같은 임베딩 모델**(text-embedding-3-small) 사용. (이미 그렇게 설정됨)

## 4. 연동 스모크 테스트 (curl)

서비스 띄운 뒤, 연결·응답 형식 확인:

```bash
curl -s -X POST http://localhost:8001/api/v1/chat \
  -H "Content-Type: application/json" \
  -H "X-Trace-Id: 11111111-1111-1111-1111-111111111111" \
  -d '{
    "roomId": 142,
    "userSeq": 12345,
    "companyContext": {"corpCode": "00126380", "corpName": "삼성전자"},
    "messages": [{"role": "user", "content": "영업이익 작년보다 얼마나 늘었어?"}]
  }'
```

**기대 응답(요약)** — `answerText`·`sources`·`verification` 채워지고 `error:null`:
```json
{
  "roomId": 142, "intent": "qa",
  "answerText": "연결재무제표 기준 영업이익은 ... 10,875,090,000,000원 증가했습니다.",
  "sourceContent": "[20260310002820 / 삼성전자 2025 사업보고서 재무제표]\n영업이익 — ...",
  "macroSnapshot": null,
  "sources": [{"rceptNo":"20260310002820","reportNm":"2025 사업보고서 재무제표", "...": "..."}],
  "outOfScope": false, "detectedCompany": null, "needsClarification": false,
  "verification": {"verdict":"pass","groundedScore":1.0},
  "error": null
}
```

**다른 회사(스코프 가드) 테스트:**
```bash
# messages 를 "현대차 영업이익은?" 으로 바꾸면 → outOfScope:true, detectedCompany:"현대차"
```

**기업 corpCode 참고:** 삼성전자 `00126380` / 현대자동차 `00164742`

---

## 5. 타임아웃·성능 메모

- 한 턴 처리 = 라우터 + 검색 + QA + 검증 (+ 거시/재무 결합, +fail 시 CRAG 재검색 1회)
- 보통 수~십수 초. **CRAG·거시 결합이 겹치면 더 걸릴 수 있음** → 명세상 **Response Timeout 90초** 권장
- LLM 호출은 내부에서 타임아웃(60초) 목표
