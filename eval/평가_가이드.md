## 1. ⚠️ 먼저 알아야 할 함정 (이거 모르면 삽질함)

1. **`expected_chunk_ids`로 매칭하지 마라.**
   골든셋의 `expected_chunk_ids`는 *만들 당시의 청킹*에서 나온 id다. 청킹 설정(800/1200)이 다르거나 재적재하면 id가 안 맞아서 전부 miss 처럼 보인다.
   → **`expected_answer`(정답 값)의 구문/숫자 매칭**으로 hit 판정해라. (청킹에 무관)

2. **정답 숫자 추출해서 매칭.**
   `expected_answer` 예: `"5,969,782,550주"`. 여기서 **가장 긴 숫자 토큰**(`5,969,782,550`)을 뽑아, 검색된 청크 본문에 그 숫자가 들어있으면 hit.
   숫자가 없는 답(예: "적정의견", "8.51%")은 `expected_terms`가 모두 들어있으면 hit 으로 처리(소수점·정성 답은 별도 케어).

3. **정규화 후 비교.** 공백 제거(`re.sub(r"\s+","",s)`) 후 substring 비교. 표는 띄어쓰기가 들쭉날쭉하다.

4. **검색은 `search_corpus` 그대로 사용** (rerank·keyword fallback 포함된 실제 검색 경로).

## 2. 러너가 할 일 (의사코드)

```python
import json, re
from app.rag.vectorstore import get_vector_store

CORP = "00126380"  # 삼성전자
COLL = f"corpus_{CORP}"
golden = json.load(open("eval/golden_samsung_3y_retrieval_v4.json", encoding="utf-8"))
store = get_vector_store()

def norm(s): return re.sub(r"\s+", "", str(s)).lower()
def mainnum(ans):
    nums = re.findall(r"[\d,]{3,}", ans or "")
    return max(nums, key=len) if nums else None

def hit_rank(q, top_k=10):
    cites = store.search_corpus(COLL, q["question"], top_k=top_k)
    num = mainnum(q.get("expected_answer"))
    terms = [norm(t) for t in eval(q.get("expected_terms") or "[]")]  # 문자열이면 파싱
    for i, c in enumerate(cites, 1):
        ct = norm(c.quote)
        if num and norm(num) in ct: return i
        if not num and terms and all(t in ct for t in terms): return i
    return 0

ranks = [hit_rank(q) for q in golden]
n = len(golden)
for k in (1, 3, 5):
    print(f"Hit@{k}: {sum(1 for r in ranks if r and r<=k)/n:.2f}")
print(f"MRR: {sum(1/r for r in ranks if r)/n:.2f}")
```

> `expected_terms`가 JSON 배열이면 그대로 list, 문자열(`"['보통주',...]"`)이면 `ast.literal_eval`로 파싱.

## 4. 해석 기준 (우리 실측값 참고)
- 800/120 + keyword fallback 기준 **정량 Hit@5 ≈ 0.65** 나왔음(채점 보수적이라 실제는 더 높음).
- 0.65가 낮아 보여도: ① 정량(숫자)만 모은 최악 케이스 ② 숫자 정확매칭이라 보수적 ③ 소수점·정성 답은 채점 누락. → 실질은 0.73+.
- 청킹·rerank·fallback 바꿔가며 이 숫자 비교하면 됨.
